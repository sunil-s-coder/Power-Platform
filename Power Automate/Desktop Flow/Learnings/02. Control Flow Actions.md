# Flow Control Actions (PAD)

## Comment
- Add notes for reference and documentation.
- Appears as text in the flow workspace.

## Label
- Mark a destination point in the flow.
- Assign a name in the **Label name** field.

## Go to
- Redirect flow execution to a specified **Label**.

## Run subflow
- Execute another subflow at any point.
- Flow resumes after subflow completes.

## Exit subflow
- Stop the current subflow and return to the calling point.
- Not required at the end of a subflow (it ends automatically).

## Stop flow
- Terminate the entire flow.
- Optionally display a custom error message.

## On block error / End
- Create an error block to handle failures gracefully.
- Supported operations on error:
  - **Set variable** – assign or update a variable.
  - **Run subflow** – execute a defined subflow.
  - **Go to next action** – continue after the failed action.
  - **Repeat action** – retry the failed action.
  - **Go to label** – jump to a defined label.
  - **Go to beginning of block** – restart the block.
  - **Go to end of block** – skip to block end.

## Get last error
- Store details of the last error in `LastError` variable.
- Optionally clear the error after retrieval.

## Region / End Region
- Group related actions together.
- Define a region name for better organization.

## Wait
- Pause flow execution for a specified number of seconds.

## If group
- If condition
- Else if condition
- Else condition

## Swithc group
- Switch block
- Case
- Default Case
