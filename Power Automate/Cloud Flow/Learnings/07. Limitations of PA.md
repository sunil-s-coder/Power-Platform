# Power Automate Limits & Throttling

## API Throttling
- **600 API calls per connection per 60 seconds**.  
- All actions count toward limits, including:
  - Connector actions
  - HTTP actions
  - Built-in actions (Compose, Initialize variable, etc.)
  - Retries and pagination requests
  - Successful and failed actions

## On-Premises Connector
- Provides secure connection between on-premises data and cloud flows via a **gateway**.  
- Performance depends on local **internet speed**.  
- **Alternative:** Microsoft Azure ExpressRoute  
  - Extends on-premises networks into Microsoft cloud via **private connection**  
  - Provides faster and more reliable network to Microsoft 365 and Azure  
  - Offers primary and backup connections for redundancy

# Data Policies

Data Loss Prevention (DLP) policies act as guardrails to help prevent users from unintentionally exposing organizational data. These policies can be scoped at either the **environment level** or the **tenant level**, providing flexibility to balance protection with productivity.

---

## Policy Scope

- **Tenant-level policies**  
  - Can apply to:
    - All environments  
    - Selected environments  
    - All environments *except* those explicitly excluded  

- **Environment-level policies**  
  - Defined for one environment at a time  

---

## Connector Classification

DLP policies enforce rules by classifying connectors into groups:

- **Business** ‚Äì Connectors in this group can only be used with other Business connectors in an app or flow.  
- **Non-Business** ‚Äì Default group for all connectors when a new policy is created.  
- **Blocked** ‚Äì Connectors in this group cannot be used at all.  

> üí° By default, all connectors are placed in the **Non-Business** group. You can move them to **Business** or **Blocked** as needed.

---

## Managing Connectors

You can manage connector groups when creating or modifying a DLP policy in the **Microsoft Power Platform admin center**.

**Example:**  
If you place **SharePoint** and **Outlook** in the Business group, and **Twitter** in the Non-Business group:
- ‚úÖ Makers can build flows using SharePoint + Outlook  
- ‚ùå Makers cannot build flows using SharePoint + Twitter  

This ensures that data cannot flow between connectors in different groups.

---

## Data Sharing Rules

- **Allowed:** Data can be shared among connectors *within the same group*.  
- **Restricted:** Data cannot be shared across connectors in *different groups*.
